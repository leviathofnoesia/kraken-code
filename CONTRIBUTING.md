# Contributing to Kraken Code

Thank you for your interest in contributing to Kraken Code! We welcome contributions from the community.

## Development Setup

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Install dependencies: `bun install`
4. Start development: `bun run dev`

## Code Style

- Follow the existing code style in the repository
- Use TypeScript for type safety
- Write clear, self-documenting code
- Add comments for complex logic
- Test your changes thoroughly

## Testing

- Run existing tests: `bun test`
- Add tests for new features
- Ensure all tests pass before submitting
- Test on multiple platforms if applicable

## Commit Conventions

- Use clear, descriptive commit messages
- Follow the commit message format: `type: brief description`
- Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
- Reference issues in commits: `Fixes #123`
- Atomic commits - one logical change per commit

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Update CHANGELOG.md if applicable
4. Submit a pull request with a clear description
5. Link related issues

## Feature Requests

- Search existing issues before creating a new one
- Clearly describe the feature and use case
- Explain why it's needed
- Consider if it fits the project scope

## Bug Reports

- Provide a clear, descriptive title
- Describe steps to reproduce
- Include expected and actual behavior
- Include environment details (OS, Bun version, etc.)

## Code of Conduct

Be respectful, constructive, and inclusive:
- Welcome newcomers and help them get started
- Focus on what is best for the community
- Show empathy towards other community members
- Accept feedback gracefully

## Getting Help

- Join our discussions on GitHub
- Read the documentation thoroughly
- Ask questions in issues or discussions

Thank you for contributing to Kraken Code!
